# Competition Brief — Playground Series S6E5: Predicting F1 Pit Stops

## Quick Facts

| Item | Value |
|------|-------|
| URL | https://www.kaggle.com/competitions/playground-series-s6e5 |
| Task | Binary classification |
| Target | `PitNextLap` — probability a driver pits on the next lap |
| ID column | `id` |
| Metric | **AUC-ROC** (area under the ROC curve; higher is better) |
| Deadline | **31 May 2026 23:59 UTC** (verify on Kaggle; ~23 days from now) |
| Submission limit | 5 per day |

## Submission Format

```
id,PitNextLap
0,0.123
1,0.456
...
```
`PitNextLap` must be a float in [0, 1] (predicted probability). Hard labels will score poorly.

## Data Files

- `train.csv` — features + `PitNextLap` target
- `test.csv` — features only (no target)
- `sample_submission.csv` — format reference

## Source Dataset

Synthetic data generated by a deep learning model trained on the **F1 Strategy Dataset for Pit Stop Prediction** (Kaggle: `aadigupta1601/f1-strategy-dataset-pit-stop-prediction`).

Key note: `Normalized_TyreLife` was **deliberately excluded** from the competition data to prevent near-perfect single-feature prediction. Raw tyre life / stint length are still present in some form.

## Feature Themes (expected)

- Tyre compound and age (raw laps on current tyre, not normalized)
- Race progression: lap number, total laps, lap percentage
- Position: race position, gap to leader, gap to car ahead/behind
- Performance: lap times, sector times, speed trap
- Pit stop history: last pit lap, number of stops made
- Driver / Team / Circuit (categorical)
- Weather / track temperature / track status (safety car, VSC)

## Strategy Notes

- AUC rewards well-calibrated probability ordering; class imbalance (most laps are non-pit laps) means calibration matters.
- CV split: stratified 5-fold on `PitNextLap` (preserves ~10-15% positive rate per fold).
- Leakage risk: any feature derived from future laps must be blocked. Time-series structure within a race means group-aware splits (by `RaceId` or `Driver+Race`) are worth testing vs simple stratified.
- Near-perfect features if present: safety-car flag + lap number ending (pit windows are deterministic in F1).
- The original dataset exclusion of `Normalized_TyreLife` suggests the generator is sensitive to tyre life — raw tyre age will be a top feature.
